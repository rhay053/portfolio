1. Introduction
The IAN CMS project focuses on implementing a content management system tailored for public institution websites. As part of a team of three developers, I am responsible for designing and implementing four core features, including survey management, popup/main slide management, content management, and statistics management. Additionally, I serve as the database manager, designing and creating the database using CUBRID 9.3, a platform commonly utilized by public institutions in Korea. The project uses Java and the eGovFramework (a Spring-based framework widely used in public institutions) for feature implementation.

2. Duration / Team Size
- Duration: Project duration TBD
- Team Size: 3 developers, 1 publisher, 1 designer, 1 PM

3. Key Features
- Survey Management
Functionality for creating, managing, and reviewing surveys on public institution websites.
- Popup/Main Slide Management
Features for managing and updating the website's popups and main slides.
- Content Management
Efficient content management and distribution capabilities.
- Statistics Management
Tools for collecting and analyzing key website statistics.
- Common Code Management
Functionality to manage common codes used throughout the system.
- Member Management
Features for managing user accounts and member information.
- System Management
Tools for managing system settings and monitoring operational status.
- Site Management
Features to configure and maintain key aspects of the website.
